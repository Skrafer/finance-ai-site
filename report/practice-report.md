# Отчет по проектной практике

## 1. Общие сведения

| Пункт | Описание |
| --- | --- |
| Название проекта | ИИ-ассистент по финансовой грамотности |
| Формат работы | Сайт-презентация цифрового продукта |
| Тип проекта | Статический веб-сайт |
| Основная тема | Управление личными финансами и финансовая грамотность |
| Репозиторий | `finance-ai-site` |

Проект посвящен созданию сайта, который представляет идею финансового ассистента. Сайт не является полноценным банковским приложением, а показывает концепцию продукта, его назначение, основные функции, примерную внутреннюю логику и состав команды.

## 2. Цель и задачи

Цель работы - подготовить понятный и законченный сайт о сервисе, который помогает пользователю лучше контролировать личные финансы.

Для достижения цели были поставлены следующие задачи:

- определить основную проблему, которую решает продукт;
- описать идею финансового ассистента простым языком;
- продумать структуру сайта и разделы навигации;
- подготовить страницы с описанием продукта, приложения и команды;
- оформить сайт в едином визуальном стиле;
- добавить блок, связанный с финансовой безопасностью;
- подготовить отчетные материалы и документацию в репозитории.

## 3. Актуальность проекта

Тема личных финансов актуальна для многих пользователей. Даже при стабильном доходе человеку бывает сложно понять, куда уходят деньги, какие расходы повторяются каждый месяц и какие привычки сильнее всего влияют на бюджет.

Частая проблема заключается не только в самих расходах, а в отсутствии понятной картины. Пользователь видит итоговый остаток, но не всегда понимает, какие действия привели к этому результату. Особенно это касается небольших покупок, подписок, комиссий и импульсивных трат.

Идея проекта состоит в том, чтобы представить сервис, который помогает собрать финансовые данные в понятный вид. Такой помощник может показывать категории расходов, предупреждать о рисках, помогать с целями накоплений и давать рекомендации без сложных таблиц.

## 4. Описание продукта

Финансовый ассистент задуман как сервис для повседневного контроля бюджета. Пользователь добавляет доходы, расходы, регулярные платежи и цели. После этого приложение показывает состояние бюджета и помогает понять, где можно сократить траты.

В проекте описаны следующие возможности:

- учет доходов и расходов;
- распределение операций по категориям;
- прогноз остатка бюджета;
- поиск регулярных платежей;
- контроль целей накоплений;
- советы по экономии;
- предупреждения о финансовых рисках;
- отдельный блок финансовой безопасности.

Главная идея продукта - не просто показать цифры, а объяснить финансовую ситуацию понятным языком.

## 5. Структура сайта

Проект состоит из трех основных HTML-страниц.

### `index.html`

Главная страница сайта. На ней раскрываются проблема, идея продукта, основные функции и ценность финансового ассистента для пользователя. Также размещены визуальные блоки, которые делают презентацию продукта более понятной.

### `app.html`

Страница с описанием внутренней логики приложения. Здесь показано, из каких модулей может состоять сервис: ввод данных, анализ, прогноз бюджета, рекомендации, цели и безопасность.

### `developers.html`

Страница команды. На ней указаны участники проекта, их роли и вклад в работу. Карточки участников сделаны интерактивными: их можно раскрывать для просмотра подробного описания.

## 6. Использованные технологии

| Технология | Назначение |
| --- | --- |
| HTML | Структура страниц сайта |
| CSS | Визуальное оформление, сетки, адаптивность |
| JavaScript | Анимации появления блоков и раскрытие карточек команды |
| Git | Контроль версий |

Проект не требует сборки, установки зависимостей или запуска сервера. Для просмотра достаточно открыть `index.html` в браузере.

## 7. Ход выполнения работы

### Этап 1. Выбор темы

Команда выбрала тему финансовой грамотности, потому что она связана с повседневными задачами пользователя. Было решено представить не абстрактный сервис, а помощника, который объясняет бюджет простым языком.

### Этап 2. Планирование структуры

После выбора темы была составлена структура сайта. Основной экран должен был сразу объяснять продукт, отдельная страница - показывать внутреннюю логику приложения, а страница команды - фиксировать вклад участников.

### Этап 3. Подготовка контента

Для страниц были подготовлены тексты о проблеме, функциях продукта, сценариях использования и безопасности. При подготовке материалов команда старалась сделать описание понятным для человека, который впервые видит проект.

### Этап 4. Верстка и оформление

Были созданы HTML-страницы, единый файл стилей и JavaScript-файл для небольшой интерактивности. Визуальная часть строилась вокруг аккуратных блоков, карточек, изображений и понятной навигации.

### Этап 5. Проверка результата

После сборки сайта была проверена структура страниц, переходы между разделами, отображение изображений и наличие отчетных материалов в репозитории.
