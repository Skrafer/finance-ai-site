# Отчет по проектной практике

## Название проекта

ИИ-ассистент по финансовой грамотности.

## Цель работы

Цель проектной практики - разработать и представить сайт продукта, который демонстрирует идею ИИ-ассистента для управления личными финансами. Сервис помогает пользователю лучше понимать расходы, планировать бюджет, ставить финансовые цели и получать понятные рекомендации.

## Актуальность проекта

Многие пользователи сталкиваются с тем, что личные финансы сложно контролировать вручную. Расходы складываются из мелких покупок, подписок, комиссий, кредитных платежей и спонтанных решений. Из-за этого человек часто видит только итоговый минус, но не понимает, где именно бюджет начал выходить из-под контроля.

ИИ-ассистент решает эту проблему через понятную аналитику и рекомендации. Он показывает категории расходов, предупреждает о рисках, помогает заранее оценить остаток бюджета и предлагает действия, которые можно выполнить сразу.

## Основные функции продукта

- Учет доходов и расходов.
- Категоризация финансовых операций.
- Прогноз остатка бюджета.
- Поиск регулярных платежей и скрытых подписок.
- Цели накоплений.
- Персональные советы по экономии.
- Предупреждения о финансовых рисках.
- Блок финансовой безопасности.

## Структура сайта

Проект реализован как статический сайт из трех основных страниц.

### Главная страница

Файл: `index.html`

На главной странице раскрываются проблема, идея продукта, основные функции и пользовательская ценность финансового ассистента. Также размещены визуальные материалы, показывающие интерфейс и сценарии использования.

### Страница приложения

Файл: `app.html`

На этой странице описана внутренняя логика продукта: ввод данных, обработка финансовой информации, ИИ-анализ, прогноз бюджета, рекомендации и блок информационной безопасности.

### Страница команды

Файл: `developers.html`

На странице команды указаны участники проекта, их роли и вклад в работу. Карточки участников раскрываются интерактивно.

## Использованные технологии

- HTML5 для структуры страниц.
- CSS3 для адаптивного оформления и визуального стиля.
- JavaScript для анимаций появления блоков и раскрытия карточек участников.
- Git и GitHub для хранения проекта.
- GitHub Pages как возможный способ публикации сайта.

## Реализация

В ходе работы был создан сайт, который выполняет роль презентации продукта. Основное внимание уделено понятной структуре, визуальной подаче и описанию функций сервиса. Проект не содержит серверной части и базы данных, поэтому его можно открыть локально в браузере или разместить на GitHub Pages.

Файлы проекта:

- `index.html`
- `app.html`
- `developers.html`
- `styles.css`
- `script.js`
- `assets/`
- `report/`
